<span data-ttu-id="a43c8-105">Een projectgebaseerde prijsopgaveregel heeft het concept van *opgenomen* onderdelen en *toerekenbare* onderdelen.</span><span class="sxs-lookup"><span data-stu-id="a43c8-105">A project-based quote line has the concept of *included* components and *chargeable* components.</span></span>

<span data-ttu-id="a43c8-106">Op opgenomen onderdelen is het volgende van toepassing:</span><span class="sxs-lookup"><span data-stu-id="a43c8-106">Included components are subject to:</span></span>

  - <span data-ttu-id="a43c8-107">Factureringsmethode en klantsplitsingen</span><span class="sxs-lookup"><span data-stu-id="a43c8-107">Billing method and customer splits</span></span>

<span data-ttu-id="a43c8-110">Een subset van de opgenomen onderdelen kan worden gemarkeerd als toerekenbaar met het veld **Factureringstype**.</span><span class="sxs-lookup"><span data-stu-id="a43c8-110">A subset of the included components can be marked as chargeable using the **Billing Type** field.</span></span> <span data-ttu-id="a43c8-111">Het veld **Factureringstype** is een optieset die de volgende waarden toestaat in de context van een prijsopgaveregel:</span><span class="sxs-lookup"><span data-stu-id="a43c8-111">The **Billing Type** field is an option-set that allows the following values in the context of a quote line:</span></span>

  - <span data-ttu-id="a43c8-113">Niet-toerekenbaar</span><span class="sxs-lookup"><span data-stu-id="a43c8-113">Non-chargeable</span></span>

<span data-ttu-id="a43c8-114">Toerekenbare onderdelen kunnen worden gedefinieerd voor taken, rollen en transactiecategorieën.</span><span class="sxs-lookup"><span data-stu-id="a43c8-114">Chargeable components can be defined on tasks, roles, and transaction categories.</span></span>

<span data-ttu-id="a43c8-115">Toerekenbaarheid wordt gedefinieerd voor de taken van een prijsopgaveregel en is van toepassing op alle transactieklassen die op die regel zijn opgenomen.</span><span class="sxs-lookup"><span data-stu-id="a43c8-115">Chargeability is defined on the tasks for a quote line and applies to all transaction classes included on that line.</span></span> <span data-ttu-id="a43c8-116">Als het veld **Taken opnemen** is ingesteld op **Geheel project** of is leeg gelaten, is het tabblad **Toerekenbare taken** niet beschikbaar.</span><span class="sxs-lookup"><span data-stu-id="a43c8-116">If the **Include Tasks** field is set to **Entire project** or left blank, the **Chargeable Tasks** tab isn't available.</span></span>

<span data-ttu-id="a43c8-119">Toerekenbaarheid die is gedefinieerd voor transactiecategorieën voor een prijsopgaveregel is alleen van toepassing op de transactieklasse **Onkosten**.</span><span class="sxs-lookup"><span data-stu-id="a43c8-119">Chargeability is defined on transaction categories for a  quote line and only applies to the **Expense** transaction class.</span></span> <span data-ttu-id="a43c8-120">Als het veld **Onkosten opnemen** is ingesteld op **Nee** op een prijsopgaveregel van een project, is het tabblad **Toerekenbare categorieën** niet beschikbaar.</span><span class="sxs-lookup"><span data-stu-id="a43c8-120">If the field, **Include Expenses** is set to **No** on a project quote line, the **Chargeable Categories** tab isn't available.</span></span>

### <a name="update-a-project-task-to-be-chargeable-or-non-chargeable"></a><span data-ttu-id="a43c8-121">Een projecttaak bijwerken als toerekenbaar of niet-toerekenbaar</span><span class="sxs-lookup"><span data-stu-id="a43c8-121">Update a project task to be chargeable or non-chargeable</span></span>

<span data-ttu-id="a43c8-122">Een projecttaak kan in de context van een projectgebaseerde prijsopgaveregel toerekenbaar of niet-toerekenbaar zijn, waardoor de volgende instellingen mogelijk zijn:</span><span class="sxs-lookup"><span data-stu-id="a43c8-122">A project task can be chargeable or non-chargeable in the context of a specific project-based quote line, which makes the following setup possible:</span></span>

<span data-ttu-id="a43c8-123">Als een projectgebaseerde prijsopgaveregel **Tijd** en de taak **T1** bevat, wordt de taak als toerekenbaar aan de prijsopgaveregel gekoppeld.</span><span class="sxs-lookup"><span data-stu-id="a43c8-123">If a project-based quote line includes **Time** and the task **T1** , the task is associated to the quote line as chargeable.</span></span> <span data-ttu-id="a43c8-124">Als er een tweede prijsopgaveregel is die **Onkosten** bevat, kunt u de **T1** -taak op de prijsopgaveregel als niet-toerekenbaar koppelen.</span><span class="sxs-lookup"><span data-stu-id="a43c8-124">If there is a second quote line that includes **Expenses** , you can associate the **T1** task on the quote line as non-chargeable.</span></span> <span data-ttu-id="a43c8-125">Het resultaat is dat alle tijd die voor de taak is geregistreerd, toerekenbaar is en dat alle onkosten die voor de taak zijn geregistreerd niet-toerekenbaar zijn.</span><span class="sxs-lookup"><span data-stu-id="a43c8-125">The result is that all time recorded on the task is chargeable and all expenses recorded on the task are non-chargeable.</span></span>

<span data-ttu-id="a43c8-126">Het factureringstype van een taak kan worden geconfigureerd op het tabblad **Toerekenbare taken** van een projectgebaseerde prijsopgaveregel door het veld **Factureringstype** op het subraster **Taken prijsopgaveregel** bij te werken.</span><span class="sxs-lookup"><span data-stu-id="a43c8-126">A task's billing type can be configured on the **Chargeable Tasks** tab of a project-based quote line by updating the **Billing Type** field on the **Quote Line Tasks** sub-grid.</span></span> <span data-ttu-id="a43c8-127">U kunt ook het factureringstype voor een projecttaak in het veld **Factureringstype** in het subraster van de taakfactureringsinstellingen van een project bijwerken waarin de prijsopgaveregels worden weergegeven die aan een taak zijn gekoppeld.</span><span class="sxs-lookup"><span data-stu-id="a43c8-127">Alternatively, you can update the billing type for a project task in the **Billing Type** field on the sub-grid on the task billing setup of a project that shows the quote lines associated to a task.</span></span>
